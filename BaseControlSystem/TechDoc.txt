超市智能导购车 下位机技术文档
================================

一、项目概述
--------------------------------

项目名称: 超市智能导购车
我的分工: 下位机 (STM32F411CEU6 嵌入式端)
  上位机和服务端由其他同学负责

系统三层架构:
  服务端  --WiFi/网络-->  上位机  --UART-->  下位机(我)
  (商品DB+Web)          (LVGL界面)            (UWB定位+底盘+雷达+导航)

下位机职责:
  1. 接收上位机发来的目标坐标 (x, y) — 上位机自行处理商品→坐标
  2. UWB 定位模块自己解算自身坐标
  3. 编码器航迹推算 + UWB修正 = 当前位姿
  4. 根据目标坐标和当前位姿，计算速度(v,w)，导航到目标
  5. 两轮 PID 差速控制底盘
  6. LD06 激光雷达避障


二、芯片和外设选型
--------------------------------

  主控:      STM32F411CEU6 (Cortex-M4, 100MHz, 512KB Flash, 128KB SRAM)
  激光雷达:  LD06_LD (乐动机器人, 0.02~12m, UART 230400bps)
  UWB 定位:  MK8000 (USART1, 230400bps, 单字节IT中断)
  电机驱动:  TB6612 双H桥
  编码器:    霍尔编码器 (电机自带, 11PPR, 90:1减速比, 轮径67.5mm)
  通信:      UART串口与上位机通信 (USART6, 115200bps)
  工具链:    Keil MDK-ARM V5.32 + CubeMX 6.17
  HAL库:     STM32Cube FW_F4 V1.28.3

八、上位机通信协议 (USART6)
--------------------------------

  帧格式: [0xA5] [CMD] [LEN] [载荷LEN字节] [XOR校验]
  XOR校验 = CMD ^ LEN ^ 载荷逐字节异或

  指令:
    0x01  载荷8B    上位机发目标坐标
            前4B: x (int32, 小端, 单位mm)
            后4B: y (int32, 小端, 单位mm)
            纯路径模式: 起点固定(0,0) → Path_Plan(0,0,x,y) → 逐条回传 0x90
            完整模式:      Navi_SetTarget(x,y) → 开始导航 + PID底盘

    0x02  载荷0B    急停 (仅 DEBUG_ENABLE_NAVI=1 时有效)

    0x04  载荷2B    开环测试 (仅 DEBUG_ENABLE_MOTOR=1 时有效)
            前1B: 左轮占空比% (uint8, 0~100)
            后1B: 右轮占空比% (uint8, 0~100)

    0x05  载荷8B    设定锚点世界坐标 (仅 DEBUG_ENABLE_COORD_SOLVER=1 时有效)
            前2B: ax, 再2B: ay, 再2B: bx, 再2B: by (int16*4, mm)

    0x80  载荷10B   下位机回复当前位姿(每2000ms自动上报)
            前4B: x (int32, 小端, 单位mm)
            中4B: y (int32, 小端, 单位mm)
            后2B: yaw (int16, 小端, 单位0.01度)

    0x90  载荷10B   下位机回复路点 (响应 0x01 目标/ASCII输入)
            前1B: 路点序号 (0-based)
            再1B: 总路点数
            再4B: x (int32, 小端, 单位mm)
            再4B: y (int32, 小端, 单位mm)
            上位机收齐 total 条即完成规划

  已删除指令(06-26): 0x06(商品目标)/0x07(地图查询)/0x81(商品回复)/0x82(地图信息)
  → 上位机自行处理商品→坐标映射, 直接发 0x01 目标坐标即可

  举例-上位机发送目标点 (300mm, 400mm):
    十六进制: A5 01 08 2C 01 00 00 90 01 00 00 B5
    校验: 01 ^ 08 ^ 2C ^ 01 ^ 00 ^ 00 ^ 90 ^ 01 ^ 00 ^ 00 = B5

  举例-开环50%占空比测试:
    十六进制: A5 04 02 32 32 06
    校验: 04 ^ 02 ^ 32 ^ 32 = 06

  举例-急停:
    十六进制: A5 02 00 02
    校验: 02 ^ 00 = 02



九、导航控制策略
--------------------------------

  输入: Navi_SetTarget(x, y) 设目标坐标
  当前位姿来源:
    主: UWB实时定位提供(x,y)绝对坐标, 无累积漂移
    **yaw: MPU6050陀螺仪 Gz积分** (不依赖编码器差速)
        兜底: UWB掉线时编码器短期接管(x,y)位置推算

  控制律 (P控制, 每10ms):
    1. dx = 目标x - 当前x
    2. dy = 目标y - 当前y
    3. 距离 dist = sqrt(dx^2 + dy^2)
    4. 目标朝向 desired_yaw = atan2(dy, dx)
    5. 角度误差 err = norm(desired_yaw - 当前yaw)  范围(-PI,PI)
    6. v = NAVI_K_DIST * dist,  限幅到 MAX_SPEED
    7. w = NAVI_K_ANGLE * err,  限幅到 MAX_OMEGA
    8. 大角度偏差(>57度)时 v *= 0.3 (先转后走)
    9. Chassis_SetTarget(v, w)
    10. dist < 500mm 判定到达, 自动停车

  可调参数:
    NAVI_ARRIVE_MM        = 500   到达判定距离(mm) — 格级导航容差
    NAVI_MAX_SPEED_MM_S   = 300   最大线速度(mm/s)
    NAVI_MAX_OMEGA        = 0.6   最大角速度(rad/s) — 防UWB噪声放大
    NAVI_K_DIST           = 0.5   距离->速度 P系数 — 略增克服起步停滞
    NAVI_K_ANGLE          = 0.6   角度差->角速度 P系数 — 降低防摆动
    NAVI_ANGLE_EMA_ALPHA  = 0.15  角度误差EMA α — 滤除UWB跳变
    NAVI_SLOW_ANGLE_THRESH= 1.5   超过86°才减速 (防UWB噪声频繁误触发)

十二A、UWB定位 (2026-06-28 重写)
--------------------------------

  算法: trilat2D_3A 线性最小二乘 (移植自 Arduino ESP32_UWB_tag2D_3A)
    三锚全在线 → trilat2D_3A
    C离线      → AB三角回退
    A离线      → BC三角回退

  trilat2D_3A 特性:
    - 自动选最近锚作参考 → d[0]²最小 → 误差传播最小
    - 近A时A作参考, 近B时B作参考 → 避免固定参考锚远距误差放大
    - A矩阵仅依赖锚点坐标, 开机算一次 (静态变量)

  锚点配置 (06-28最终):
    UWB局部系: A(8.88,0)  B(0.2,0)  C(9.66,8.93)
    世界系:     A(8880,0)  B(0,0)    C(9660,8930)
    物理模块:   A=0x0002   B=0x0001  C=0x0003

  高度 (06-28实测修正):
    标签=0.52m  锚点=1.90m (三锚等高) → dz=1.38m

  天线延迟 (运行时变量, USART6串口实时可调):
    A = 0.50m   B = 1.40m   C = 0.00m

  数据有效性门限:
    锚点超时: 500ms无新数据 → 定位无效
    RSSI最低: 30 (过低丢弃)
    斜距<0.2m或>30m丢弃

  滤波: 斜距EMA α=0.15 (每锚独立)

  printf格式 (每1s):
    X=%.2f Y=%.2f eA=%.2f eB=%.2f eC=%.2f Q=%d A=%c B=%c C=%c R=%.3f d=%.2f/%.2f/%.2f

  串口实时调延 (USART6 ASCII):
    A0.5 回车 → 设A延迟0.50m
    B1.4 回车 → 设B延迟1.40m
    ? 回车   → 查看当前延迟值


十二B、坐标解算器 (2026-06-20, B原点)
--------------------------------

  目标: UWB局部坐标 → 超市世界坐标 (B=(0,0), +X=西, +Y=北)

  变换步骤:
    1. 计算AB连线在世界系中的方向角 θ = atan2(By-Ay, Bx-Ax)
    2. 旋转: X' = xL·cosθ - yL·sinθ,  Y' = xL·sinθ + yL·cosθ
    3. 平移: Xw = Ax + X',  Yw = Ay + Y'

  锚点世界坐标: 由上位机CMD 0x05下发 (int16×4, mm)
  UWB局部间距: UWB_ANCHOR_SPACING_M = 8.88m (用户实测)

  锚A=0x0002(西南) B=0x0001(东南) C=0x0003(西北)
  定位: trilat2D_3A 线性最小二乘(自动近锚参考) + AB/BC回退
  锚间距: AB=8.68m, 标签0.52m 锚1.90m → dz=1.38m

雷达测试 (LD06, USART2 230400):
  硬件: PA2→LD06_RX, PA3→LD06_TX, PB8→电机PWM(30kHz 50%)
  供电: 雷达独立5V
  上电后听电机转声 → 2~3s初始化 → 串口输出 "#1800" 完整圈帧
  数据格式: 角度(°),距离(cm),强度  (CSV, 每40点抽1, 仅≥1600点完整圈)
  ⚠️ DMA+IDLE链路已验证
  ⚠️ 雷达帧需过滤半圈残帧(count<1600丢弃), 整帧buf一次性HAL_UART_Transmit发送


纯路径规划模式 (06-26 新增):
  1. debug_config.h: LED=1, PATH_PLAN=1, HOST_UART=1, 其余=0
  2. 上电无任何硬件需求 (电机/雷达/UWB全关), LED 1Hz心跳
  3. 上位机发目标 (0x01, 8B) → Path_Plan(0,0,x,y) → 逐条回传 0x90 路点
  4. 也可 ASCII 文本交互: 串口直输 "x y"(m) → 回传路点序列

全功能自主导航模式 (06-26 新增):
  1. debug_config.h: 开 LED/UWB/ENCODER/MOTOR/PID/NAVI/COORD/MAP/PATH/HOST
  2. 上电流程:
     App_Init → 设锚点(8880,0)-(0,0) → 3s等UWB稳定 → 目标(9270,4230)mm
     → 2s等位姿收敛 → Chassis_PID_Enable(1) → 发车!
  3. 小车自动: UWB定位 → yaw编码器 → 路径A*规划 → PID差速 → 路点跟随
  4. 目标硬编码在 main.c: Navi_SetTarget(9270, 4230)  ← 主过道中央
  5. 上位机可在行驶中发 0x01 覆盖目标, 发 0x02 急停
  6. 所有模块受 #if DEBUG_xxx 守卫, 可随时降级回纯路径规划模式
